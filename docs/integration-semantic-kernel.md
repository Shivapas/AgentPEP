# AgentPEP — Semantic Kernel Integration Guide

## Overview

AgentPEP integrates with [Semantic Kernel](https://learn.microsoft.com/semantic-kernel/)
through the **function invocation filter pipeline**.  Every plugin function
invocation is intercepted and checked against the AgentPEP Intercept API
before the function executes.

Key capabilities:

- **Filter pipeline integration** — `AgentPEPFunctionFilter` implements the SK filter pattern
- **Plugin metadata mapping** — `SKPluginMapper` converts SK plugin/function hierarchy to flat tool names
- **Async and sync** — both `AgentPEPFunctionFilter` (async) and `AgentPEPFunctionFilterSync` variants
- **Plugin aliasing** — shorten verbose plugin names for cleaner policy rules

## Installation

```bash
pip install agentpep-sdk[semantic-kernel]
```

## Quick Start

### 1. Register the Filter on a Kernel

```python
from semantic_kernel import Kernel
from semantic_kernel.filters.filter_types import FilterTypes
from agentpep import AgentPEPClient
from agentpep.integrations.semantic_kernel import AgentPEPFunctionFilter

kernel = Kernel()
client = AgentPEPClient(base_url="http://localhost:8000", api_key="...")

policy_filter = AgentPEPFunctionFilter(
    client=client,
    agent_id="sk-agent",
    session_id="session-123",
)
kernel.add_filter(FilterTypes.FUNCTION_INVOCATION, policy_filter)
```

Every subsequent plugin function invocation on this kernel will be checked
against AgentPEP.  If the policy decision is `DENY` or `ESCALATE`, a
`PolicyDeniedError` is raised and the function never executes.

### 2. Use with Plugins

```python
from semantic_kernel.functions import kernel_function

class MathPlugin:
    @kernel_function(description="Add two numbers")
    def add(self, a: int, b: int) -> int:
        return a + b

    @kernel_function(description="Multiply two numbers")
    def multiply(self, a: int, b: int) -> int:
        return a * b

kernel.add_plugin(MathPlugin(), plugin_name="MathPlugin")

# This invocation is now policy-checked:
result = await kernel.invoke("MathPlugin", "add", a=5, b=3)
```

The tool name sent to AgentPEP will be `"MathPlugin.add"`.

## Plugin Metadata Mapping (APEP-169)

Semantic Kernel organises functions within named plugins.  `SKPluginMapper`
converts this two-level hierarchy into flat tool names for AgentPEP.

### Default Behaviour

```python
from agentpep.integrations.semantic_kernel import SKPluginMapper

mapper = SKPluginMapper()
mapper.tool_name("MathPlugin", "add")      # => "MathPlugin.add"
mapper.tool_name(None, "standalone_func")   # => "standalone_func"
```

### Plugin Aliases

Shorten verbose plugin names for cleaner policy rules:

```python
mapper = SKPluginMapper(
    plugin_alias={
        "MathPlugin": "math",
        "FileAccessPlugin": "file",
        "EmailPlugin": "email",
    },
)
mapper.tool_name("MathPlugin", "add")       # => "math.add"
mapper.tool_name("FileAccessPlugin", "read") # => "file.read"
```

### Custom Separator

```python
mapper = SKPluginMapper(separator="/")
mapper.tool_name("MathPlugin", "add")  # => "MathPlugin/add"
```

### Extract Metadata

Extract function metadata for documentation or policy authoring:

```python
metadata = mapper.extract_metadata(kernel_function)
# {
#     "name": "add",
#     "plugin_name": "MathPlugin",
#     "description": "Add two numbers",
#     "parameters": [
#         {"name": "a", "description": "", "type": "int", "required": True},
#         {"name": "b", "description": "", "type": "int", "required": True},
#     ]
# }
```

## Using with the Mapper

Pass a custom mapper to the filter for alias resolution:

```python
mapper = SKPluginMapper(
    plugin_alias={"MathPlugin": "math", "EmailPlugin": "email"},
)

policy_filter = AgentPEPFunctionFilter(
    client=client,
    agent_id="sk-agent",
    plugin_mapper=mapper,
)
kernel.add_filter(FilterTypes.FUNCTION_INVOCATION, policy_filter)

# Now "MathPlugin.add" is sent as "math.add" to AgentPEP
```

## Synchronous Usage

For non-async environments, use `AgentPEPFunctionFilterSync`:

```python
from agentpep.integrations.semantic_kernel import AgentPEPFunctionFilterSync

sync_filter = AgentPEPFunctionFilterSync(
    client=client,
    agent_id="sk-agent",
)
```

## Error Handling

```python
from agentpep.exceptions import PolicyDeniedError

try:
    result = await kernel.invoke("AdminPlugin", "delete_all")
except PolicyDeniedError as e:
    print(f"Function '{e.tool_name}' denied: {e.reason}")
```

## Policy Rule Examples

Create policy rules that match the tool names generated by `SKPluginMapper`:

| Rule Pattern | Matches | Action |
|---|---|---|
| `math.*` | All MathPlugin functions | ALLOW |
| `email.send` | Only the send function | ESCALATE |
| `AdminPlugin.*` | All admin functions | DENY |
| `file.read` | File read only | ALLOW |
| `file.delete` | File delete | DENY |

## Configuration Summary

### AgentPEPFunctionFilter

| Parameter | Default | Description |
|-----------|---------|-------------|
| `client` | — | `AgentPEPClient` instance |
| `agent_id` | — | Agent ID for policy evaluation |
| `session_id` | `"default"` | Session identifier |
| `plugin_mapper` | `SKPluginMapper()` | Custom plugin-to-tool-name mapper |

### SKPluginMapper

| Parameter | Default | Description |
|-----------|---------|-------------|
| `separator` | `"."` | Character joining plugin and function names |
| `plugin_alias` | `None` | Map from SK plugin names to shorter aliases |
