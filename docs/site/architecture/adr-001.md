# ADR-001: Monorepo Structure

## Status

Accepted

## Context

AgentPEP consists of multiple components: backend API, frontend console,
Python SDK, and infrastructure configs. We need a repository structure that
supports independent versioning and parallel development.

## Decision

Use a monorepo with top-level directories for each component:

```
agentpep/
├── backend/    (Python/FastAPI)
├── frontend/   (React/Vite)
├── sdk/        (Python package)
├── infra/      (Terraform/Helm)
└── docs/       (Documentation)
```

## Consequences

- Single clone for full development environment
- Shared CI pipeline with per-component jobs
- Independent versioning (SDK follows semver, backend follows release tags)
- Clear ownership boundaries between components
